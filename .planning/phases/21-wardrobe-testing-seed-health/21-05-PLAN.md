**Story ID format:**
- Storybook auto-generates IDs as kebab-case of `meta.title` joined by `--` then kebab-case of story export name.
- Example: `meta.title = "Wardrobe/Catalog/DressCard"` + `export const Default: Story = ...` → ID `wardrobe-catalog-dresscard--default`.
- Verify each new ID matches Storybook's actual emitted ID by running `pnpm storybook` and inspecting the URL bar OR by running `pnpm storybook:build` and grepping `storybook-static/stories.json`.

**MSW handler pattern (canonical):**
- `http.get("*/api/trpc/<router>.<procedure>*", () => HttpResponse.json([{ result: { data: { json: {...} } } }]))` — the `*` prefix matches batched URL encoding, the `[...]` array wrapping handles batched response shape.
- For procedures returning Date/BigInt/null, use `data: { json: {...}, meta: {} }` (superjson serialize shape). For pure scalars/strings/Booleans, `data: {...}` direct works.
- Reference: SmartKPICards.stories.tsx (the only working msw-wrapped story today).

**4-story convention (Default, Empty, Loading, +1):**
- Every story file ships AT LEAST 3 stories: `Default`, `Empty`, `Loading`. Add a 4th when it visually differs meaningfully (e.g., HighVolume, MultipleStates).
- Loading state via `parameters.msw.handlers: [http.get(..., async () => { await new Promise(() => {}); })]` — keeps the TRPC query in-flight forever, rendering the component's skeleton/spinner.
