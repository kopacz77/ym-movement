  <action>
    Create `src/features/admin/api/queries/superAdminQueries.ts` with three queries:

    **1. getCoachesOverview** (SADM-01)
    - Uses `superAdminProcedure` with no input
    - Fetches all approved coaches with:
      - User info (name, email) via `include: { User: { select: { name: true, email: true } } }`
      - Counts: `_count: { select: { Lesson: true, CoachStudent: true, RinkTimeSlot: { where: { isActive: true } } } }`
    - For each coach, run parallel aggregation queries:
      - Total hours booked: `prisma.lesson.aggregate({ where: { coachId: coach.id, status: "SCHEDULED" }, _sum: { duration: true } })` -- convert from minutes to hours (divide by 60)
      - Current month earnings: `prisma.payment.aggregate({ where: { Lesson: { coachId: coach.id }, status: "COMPLETED", lesson_date: { gte: startOfMonth } }, _sum: { amount: true } })` -- note Payment scoping through Lesson relation
    - Return array of coach objects with: id, name, email, isActive, suspendedAt, revenueSplitPercent, lessonCount, studentCount, activeSlots, totalHoursBooked, monthEarnings (after split), totalRevenue (before split)
    - Use `Promise.all` for the per-coach earnings queries to parallelize

    **2. getCoachDetail** (SADM-02)
    - Uses `superAdminProcedure`
    - Input: `z.object({ coachId: z.string() })`
    - Fetches the specific coach with full profile data:
      - `User: { select: { name, email, image } }`
      - `bio`, `skills`, `certifications`, `isActive`, `isApproved`, `suspendedAt`, `revenueSplitPercent`
    - Fetches upcoming lessons for this coach: `prisma.lesson.findMany({ where: { coachId, status: "SCHEDULED", startTime: { gte: new Date() } }, include: { Student: { include: { User: { select: { name: true } } } }, Rink: { select: { name: true } } }, orderBy: { startTime: "asc" }, take: 20 })`
    - Fetches student roster: `prisma.coachStudent.findMany({ where: { coachId }, include: { Student: { include: { User: { select: { name: true, email: true } } } } } })`
    - Fetches this month's lesson count and earnings (same aggregation pattern as overview but for single coach)
    - Returns combined object with coach profile, upcoming lessons, student roster, and stats

    **3. getRevenueBreakdown** (SADM-03)
    - Uses `superAdminProcedure`
    - Input: `z.object({ startDate: z.date().optional(), endDate: z.date().optional() })`
    - Defaults: startDate = start of current month, endDate = now
    - Fetches all active coaches with their User info
    - For each coach, aggregates:
      - Total revenue: `prisma.payment.aggregate({ where: { Lesson: { coachId: coach.id }, status: "COMPLETED", lesson_date: { gte: startDate, lte: endDate } }, _sum: { amount: true } })`
      - Lesson count in period: `prisma.lesson.count({ where: { coachId: coach.id, startTime: { gte: startDate, lte: endDate } } })`
    - Calculates per-coach: coachPayout = totalRevenue * (revenueSplitPercent / 100), platformRevenue = totalRevenue - coachPayout
    - Also calculates platform totals: sum of all totalRevenue, sum of all coachPayouts, sum of all platformRevenue
    - Returns: `{ coaches: [...perCoachData], totals: { totalRevenue, totalCoachPayouts, totalPlatformRevenue } }`

    **Import pattern:** `import { z } from "zod"; import { createTRPCRouter, superAdminProcedure } from "@/lib/trpc";`

    **Export:** `export const superAdminDashboardRouter = createTRPCRouter({ ... })`

    **CRITICAL:** Payment has NO direct coachId column. Always scope through Lesson relation: `where: { Lesson: { coachId } }`. The research warns about this as Pitfall 1.
