1. Create `src/features/student/api/queries/coachBrowseQueries.ts`:
   - `getBrowsableCoaches`: protectedProcedure (NOT admin/superAdmin -- students call this). Query `prisma.coach.findMany` where `isApproved: true, isActive: true`. Include `User: { select: { name: true, email: true } }` and `_count: { select: { RinkTimeSlot: { where: { isActive: true, startTime: { gte: new Date() } } } } }`. Map results to return: id, name (from User.name), bio, photoUrl, skills, certifications, yearsExperience, privateLessonPrice, groupLessonPrice, choreographyPrice, competitionPrepPrice, availableSlots (from _count.RinkTimeSlot). Do NOT expose revenueSplitPercent, googleCalendarId, tokens, or suspendedReason. Order by `createdAt: "asc"`.
   - `getCoachProfile`: protectedProcedure with input `z.object({ coachId: z.string() })`. Same query but for a single coach by id, also filtered by `isApproved: true, isActive: true`. Return same fields plus full bio. Throw NOT_FOUND if coach doesn't exist or isn't active/approved.
   - Export as `coachBrowseRouter`.

2. Register in `src/features/student/api/queries/index.ts`:
   - Import `coachBrowseRouter` from `./coachBrowseQueries`
   - Add `coachBrowse: coachBrowseRouter` to the studentRouter. Name `coachBrowse` avoids collision with any existing router names.

3. Update `src/features/student/api/queries/availabilityQueries.ts`:
   - Add `coachId: z.string().optional()` to the input schema of `getAvailableTimeSlots`.
   - In the WhereClause interface, add `coachId?: string`.
   - After the rinkId filter block, add: `if (input.coachId) { whereClause.coachId = input.coachId; }`.
   - Add `Coach: { select: { id: true, User: { select: { name: true } } } }` to the include alongside existing Rink and Lesson includes.

4. Update `src/lib/pricing.ts`:
   - Add optional `coach` parameter (LAST parameter) to `getHourlyRateForLessonType`:

     ```
       choreographyPrice?: number | null;
       competitionPrepPrice?: number | null;
     } | null
     ```
   - Insert coach pricing check BETWEEN the student custom pricing block and the default pricing block. Same switch pattern as student custom pricing: check `coach.privateLessonPrice != null` for PRIVATE, etc. Only apply if coach param is provided and the specific price is non-null.
   - Add the same `coach` parameter to `calculateLessonPrice` and pass it through to `getHourlyRateForLessonType`. Make it the LAST parameter and optional so all existing callers continue to work without changes.
