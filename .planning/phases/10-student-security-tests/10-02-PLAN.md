    **File structure:**
    ```typescript
    import { test, expect } from "@playwright/test";
    ```

    **SECT-02: Role Guard Enforcement (3 nested describe blocks):**

    1. "student cannot access admin or coach routes" -- `test.use({ storageState: "playwright/.auth/student.json" })`

       a. "student redirected from /admin/dashboard to /student/dashboard"
          - `await page.goto("/admin/dashboard")`
          - `await expect(page).toHaveURL(/\/student\/dashboard/, { timeout: 10000 })`

       b. "student redirected from /coach/dashboard to /student/dashboard"
          - `await page.goto("/coach/dashboard")`
          - `await expect(page).toHaveURL(/\/student\/dashboard/, { timeout: 10000 })`

    2. "coach cannot access admin routes" -- `test.use({ storageState: "playwright/.auth/coach.json" })`

       a. "coach redirected from /admin/dashboard to /coach/dashboard"
          - `await page.goto("/admin/dashboard")`
          - `await expect(page).toHaveURL(/\/coach\/dashboard/, { timeout: 10000 })`

    3. "unauthenticated user redirected to login" -- `test.use({ storageState: { cookies: [], origins: [] } })`

       a. "unauthenticated user redirected from /admin/dashboard to login"
          - `await page.goto("/admin/dashboard")`
          - `await expect(page).toHaveURL(/\/auth\/login/, { timeout: 10000 })`

       b. "unauthenticated user redirected from /student/dashboard to login"
          - `await page.goto("/student/dashboard")`
          - `await expect(page).toHaveURL(/\/auth\/login/, { timeout: 10000 })`

       c. "unauthenticated user redirected from /coach/dashboard to login"
          - `await page.goto("/coach/dashboard")`
          - `await expect(page).toHaveURL(/\/auth\/login/, { timeout: 10000 })`

    **SECT-03: Dual-Role Navigation (1 describe block):**

    Use `test.use({ storageState: "playwright/.auth/super-admin.json" })` -- the SUPER_ADMIN user has a Coach record in the seed data, so the sidebar will show dual-role links.

    1. "admin can switch to coach view and back"
       - Navigate to /admin/dashboard
       - Wait for dashboard heading to be visible
       - Find "Coach View" link in the sidebar: `page.locator('a:has-text("Coach View")')`
       - Verify it is visible (timeout: 10000)
       - Click the link
       - Wait for URL to be /coach/dashboard (timeout: 10000)
       - Verify "Coach Dashboard" heading appears
       - Find "Admin View" link in the coach sidebar: `page.locator('a:has-text("Admin View")')`
       - Verify it is visible
       - Click to return to admin
       - Wait for URL to be /admin/dashboard (timeout: 10000)
       - Verify admin dashboard heading reappears

    - Use regex patterns in toHaveURL to handle potential query params or trailing slashes.

    - For unauthenticated tests, use `{ cookies: [], origins: [] }` as storageState (established pattern from coach-flows.spec.ts line 6).
