**The five procedures:**

1. **`list`** — `adminProcedure.input(z.object({ statuses: z.array(z.nativeEnum(DressStatus)).optional(), search: z.string().optional(), page: z.number().int().positive().default(1), limit: z.number().int().positive().max(100).default(20) })).query(...)`. Builds a Prisma `where` clause:
   - `status: { in: input.statuses }` when statuses array provided and non-empty (otherwise return ALL statuses including PENDING_APPROVAL/REJECTED/ARCHIVED per ADMIN-01).
   - `OR: [{ title: { contains: search, mode: 'insensitive' } }, { description: { contains: search, mode: 'insensitive' } }]` when `search` is provided.
   - Pagination via `skip: (page-1)*limit, take: limit`.
   - `include: { Owner: { select: { id: true, name: true, email: true } }, Images: { where: { isPrimary: true }, select: { url: true }, take: 1 }, _count: { select: { Images: true, Requests: true, Rentals: true } } }`.
   - `orderBy: { updatedAt: 'desc' }`.
   - Return `{ dresses: <array>, total: <count>, page, limit }`. Do a parallel `ctx.prisma.dress.count({ where })` so the UI can render pagination.

2. **`byId`** — `adminProcedure.input(z.object({ id: z.string().cuid() })).query(...)`. `ctx.prisma.dress.findUnique({ where: { id: input.id }, include: { Owner: { select: { id: true, name: true, email: true } }, Images: { orderBy: { sortOrder: 'asc' } } } })`. Throw `TRPCError({ code: 'NOT_FOUND', message: 'Dress not found' })` when null.

3. **`create`** — `adminProcedure.input(dressInputSchema).mutation(...)`. `ctx.prisma.dress.create({ data: { ...input, ownerId: ctx.session.user.id, status: 'AVAILABLE' } })`. **Critical:** hardcode `status: 'AVAILABLE'` per ADMIN-02 — admins bypass the approval queue. Return the new dress (include nothing extra; the form will redirect to `/admin/wardrobe/[id]/edit` which then calls `byId`).

4. **`update`** — `adminProcedure.input(dressInputSchema.partial().extend({ id: z.string().cuid(), status: z.nativeEnum(DressStatus).optional(), ownerId: z.string().cuid().optional() })).mutation(...)`. Extract `id` from `input`, pass the rest as `data` to `ctx.prisma.dress.update({ where: { id }, data })`. Admins can flip status (ADMIN-03) and reassign ownership.

5. **`archive`** — `adminProcedure.input(z.object({ id: z.string().cuid() })).mutation(...)`. `ctx.prisma.dress.update({ where: { id: input.id }, data: { status: 'ARCHIVED', archivedAt: new Date() } })`. **No hard delete.** Audit trail preserved.

**Error handling:** Wrap each procedure body in `try / catch (error) { console.error(...); throw new TRPCError({ code: 'INTERNAL_SERVER_ERROR', message: '<verb> failed', cause: error }); }` mirroring `wardrobeSettingsQueries.ts` lines 70-94. `findUnique` returning null becomes a `NOT_FOUND` not `INTERNAL_SERVER_ERROR` — handle the null branch explicitly before the try/catch wrap, or check `error instanceof TRPCError && throw error`.

**Prisma relation casing reminder (CLAUDE.md):** `dress.Owner`, `dress.Images`, `dress._count.Images` — PascalCase. NEVER lowercase `dress.owner`.

```ts
// scripts/smoke-wardrobe-admin.ts
import { PrismaClient } from "@prisma/client";

const prisma = new PrismaClient();

async function main() {
  // Find a real admin user to own the test dress
  const admin = await prisma.user.findFirst({ where: { role: { in: ["ADMIN", "SUPER_ADMIN"] } } });
  if (!admin) throw new Error("No admin user found in dev DB; create one first");

  // 1. CREATE
  const dress = await prisma.dress.create({
    data: {
      ownerId: admin.id,
      title: "SMOKE TEST Dress",
      description: "Disposable test row from Phase 14 smoke",
      category: "CLASSICAL",
      themeTags: ["test"],
      color: "Black",
      secondaryColors: [],
      condition: "NEW",
      sizeLabel: "Adult Medium",
      status: "AVAILABLE",
    },
  });
  console.log("CREATE OK", dress.id, "status:", dress.status);

  // 2. BY ID
  const found = await prisma.dress.findUnique({
    where: { id: dress.id },
    include: {
      Owner: { select: { id: true, name: true, email: true } },
      Images: { orderBy: { sortOrder: "asc" } },
    },
  });
  if (!found) throw new Error("byId returned null");
  console.log("BY_ID OK", found.id, "Owner:", found.Owner.email);

  // 3. UPDATE — flip status, change a price
  const updated = await prisma.dress.update({
    where: { id: dress.id },
    data: { competitionPrice: 7500, internalNotes: "smoke-test note" },
  });
  console.log("UPDATE OK", updated.competitionPrice, updated.internalNotes);

  // 4. LIST — with status filter
  const listResult = await prisma.dress.findMany({
    where: { status: { in: ["AVAILABLE"] } },
    include: {
      Owner: { select: { id: true, name: true, email: true } },
      Images: { where: { isPrimary: true }, select: { url: true }, take: 1 },
      _count: { select: { Images: true, Requests: true, Rentals: true } },
    },
    orderBy: { updatedAt: "desc" },
    take: 5,
  });
  const total = await prisma.dress.count({ where: { status: { in: ["AVAILABLE"] } } });
  console.log("LIST OK", listResult.length, "of", total, "AVAILABLE dresses");

  // 5. ARCHIVE — soft delete
  const archived = await prisma.dress.update({
    where: { id: dress.id },
    data: { status: "ARCHIVED", archivedAt: new Date() },
  });
  console.log("ARCHIVE OK", archived.status, archived.archivedAt);

  // CLEANUP — hard delete the test row (smoke only; production never hard-deletes)
  await prisma.dress.delete({ where: { id: dress.id } });
  console.log("CLEANUP OK row gone");
}

main()
  .then(() => process.exit(0))
  .catch((e) => {
    console.error(e);
    process.exit(1);
  });
```
