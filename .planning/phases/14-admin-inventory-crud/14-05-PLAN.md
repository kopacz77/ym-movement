**URL state parsing helpers:**
```tsx
const PAGE_SIZE = 20;
const ALL_STATUSES: DressStatus[] = [
  "AVAILABLE",
  "PENDING_APPROVAL",
  "PENDING",
  "RENTED",
  "MAINTENANCE",
  "REJECTED",
  "ARCHIVED",
];

function parseStatusesParam(raw: string | null): DressStatus[] {
  if (!raw) return ["AVAILABLE"];
  const parts = raw.split(",").filter((s) => ALL_STATUSES.includes(s as DressStatus)) as DressStatus[];
  return parts.length > 0 ? parts : ["AVAILABLE"];
}
```

**Component body:**
```tsx
export function DressInventoryGrid() {
  const router = useRouter();
  const searchParams = useSearchParams();
  const utils = api.useUtils();

  // Read URL state
  const statuses = useMemo(() => parseStatusesParam(searchParams.get("statuses")), [searchParams]);
  const search = searchParams.get("q") ?? "";
  const page = Number.parseInt(searchParams.get("page") ?? "1", 10) || 1;

  // Search input — debounced via local state so typing doesn't thrash URL
  const [searchInput, setSearchInput] = useState(search);

  // Sync URL params helper
  const updateParams = (next: { statuses?: DressStatus[]; q?: string; page?: number }) => {
    const params = new URLSearchParams(searchParams.toString());
    if (next.statuses !== undefined) {
      if (next.statuses.length === 0 || (next.statuses.length === 1 && next.statuses[0] === "AVAILABLE")) {
        params.delete("statuses");
      } else {
        params.set("statuses", next.statuses.join(","));
      }
      params.delete("page");  // reset page on filter change
    }
    if (next.q !== undefined) {
      if (next.q === "") params.delete("q"); else params.set("q", next.q);
      params.delete("page");
    }
    if (next.page !== undefined) {
      if (next.page <= 1) params.delete("page"); else params.set("page", String(next.page));
    }
    router.replace(`?${params.toString()}`, { scroll: false });
  };

  // Query — uses URL state directly so refresh preserves filters
  const { data, isLoading, isFetching } = api.admin.wardrobe.list.useQuery({
    statuses: statuses.length === ALL_STATUSES.length ? undefined : statuses,
    search: search || undefined,
    page,
    limit: PAGE_SIZE,
  });

  const archive = api.admin.wardrobe.archive.useMutation({
    onSuccess: () => {
      utils.admin.wardrobe.list.invalidate();
      toast.success("Dress archived");
    },
    onError: (e) => toast.error("Failed to archive", { description: e.message }),
  });

  const handleArchive = (id: string, title: string) => {
    showDeleteConfirmation(`"${title}"`, () => archive.mutate({ id }));
  };

  // Search submit (Enter key or button) flushes to URL
  const handleSearchSubmit = (e: React.FormEvent) => {
    e.preventDefault();
    updateParams({ q: searchInput.trim() });
  };

  const total = data?.total ?? 0;
  const dresses = data?.dresses ?? [];
  const totalPages = Math.max(1, Math.ceil(total / PAGE_SIZE));
```

**Critical patterns honored:**
- **URL-state filtering** — selected statuses, search, and page all encoded in query params (`?statuses=AVAILABLE,RENTED&q=red&page=2`). Refresh-survivable, shareable. Special-case: when selection is exactly `["AVAILABLE"]` (the default) the param is dropped to keep the URL clean.
- **Default selection is `["AVAILABLE"]`** — admins typically want to see what's live first. Empty selection is impossible to reach via this UI (parseStatusesParam falls back to AVAILABLE).
- **All-status case** — when all 7 statuses are selected, the query sends `statuses: undefined` (server returns ALL rows). Cheaper than sending the full array.
- **Archive action** uses `showDeleteConfirmation` (NOT `window.confirm`). The "delete-flavored" UI deliberately reads as archive in copy ("Dress archived" toast) — there's no hard delete in MVP.
- **Archived rows** render with opacity-60 and no archive button (you can't archive what's already archived).
- **Pagination** uses `total / limit` ceil. Disable Prev on page 1, Next on last page.
- **Prisma relations** all PascalCase: `dress.Owner.email`, `dress.Images`, `dress._count.Images`. NEVER lowercase.
- **Brand:** cyan #0891b2 CTAs, navy #1a3a5c headings, slate borders, rose for destructive ghost.
- **Card shadow:** standardized `shadow-[0_1px_3px_rgba(0,0,0,0.04),0_8px_32px_rgba(0,0,0,0.02)]` per CLAUDE.md.
- **Radix focus rule:** Archive button uses `focus:text-rose-700 focus:bg-rose-50` not `hover:` (per CLAUDE.md — but this is a plain Button not DropdownMenuItem; either works, but mirroring the convention).
