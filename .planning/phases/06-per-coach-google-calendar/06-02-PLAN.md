<task type="auto">
  <name>Task 1: Create OAuth initiation and callback API routes</name>
  <files>src/app/api/auth/google-calendar/route.ts, src/app/api/auth/google-calendar/callback/route.ts</files>
  <action>
1. Create `src/app/api/auth/google-calendar/route.ts` (OAuth initiation):
   - Export async `GET` function
   - Get server session via `getServerSession(authOptions)` from `@/lib/auth`
   - If no session or no user id, return 401 JSON response
   - Look up Coach record: `prisma.coach.findUnique({ where: { userId: session.user.id } })`
   - If no coach record, return 403 JSON response with "Coach profile not found"
   - Call `generateAuthUrl(coach.id)` from `@/lib/google/oauth`
   - Return `NextResponse.redirect(url)`
   - Import prisma from `@/lib/prisma`

2. Create `src/app/api/auth/google-calendar/callback/route.ts` (OAuth callback):
   - Export async `GET(req: Request)` function
   - Parse URL params: `code`, `state` (coachId), `error`
   - If error or missing code/state, redirect to `/coach/profile?calendar=error`
   - **CSRF validation (Research Pitfall 3):** Get server session, verify the logged-in user's Coach record id matches the `state` parameter. If mismatch, redirect to `/coach/profile?calendar=error`
   - Exchange code for tokens: `createOAuth2Client()` then `oauth2Client.getToken(code)`
   - Set credentials on client, query `calendarList.get({ calendarId: "primary" })` to get the coach's primary calendar ID
   - Store encrypted tokens on Coach record:
     ```
     prisma.coach.update({
       where: { id: state },
       data: {
         googleAccessToken: encrypt(tokens.access_token!),
         googleRefreshToken: encrypt(tokens.refresh_token!),
         googleTokenExpiresAt: tokens.expiry_date ? new Date(tokens.expiry_date) : null,
         googleCalendarId: calendarId,
       },
     })
     ```
   - Redirect to `/coach/profile?calendar=connected`
   - Wrap the entire token exchange + storage in try/catch. On error, log it and redirect to `/coach/profile?calendar=error`
   - Import: `google` from `googleapis`, `createOAuth2Client` from `@/lib/google/oauth`, `encrypt` from `@/lib/encryption`, `prisma` from `@/lib/prisma`, `getServerSession` from `next-auth`, `authOptions` from `@/lib/auth`

Use `NextResponse.redirect(new URL("/coach/profile?calendar=...", req.url))` for redirects to preserve the origin.
  </action>
  <verify>Run `pnpm type-check` -- both route files should compile. Verify file structure: `ls src/app/api/auth/google-calendar/route.ts src/app/api/auth/google-calendar/callback/route.ts` both exist.</verify>
  <done>GET /api/auth/google-calendar returns a redirect to Google's consent screen for authenticated coaches. GET /api/auth/google-calendar/callback exchanges the authorization code, encrypts and stores tokens, and redirects to the profile page with a success/error status parameter.</done>

<task type="auto">
  <name>Task 2: Refactor calendar.ts from singleton to per-coach pattern</name>
  <files>src/lib/google/calendar.ts</files>
  <action>
Completely rewrite `src/lib/google/calendar.ts` to use per-coach OAuth2 instead of the service account singleton.

1. Remove ALL old code: the `JWT` import from `google-auth-library`, `getAuthClient()`, `getCalendarApi()`, service account credentials, `GOOGLE_CLIENT_EMAIL`, `GOOGLE_PRIVATE_KEY` references, `GOOGLE_CALENDAR_ID` env var usage.

2. Define `CoachWithTokens` type:
   ```typescript
   export type CoachWithTokens = {
     id: string;
     googleAccessToken: string | null;
     googleRefreshToken: string | null;
     googleTokenExpiresAt: Date | null;
     googleCalendarId: string | null;
   };
   ```

3. Create `getCoachCalendarApi(coach: CoachWithTokens)` internal function:
   - If `coach.googleAccessToken` or `coach.googleRefreshToken` is null, return null (graceful degradation)
   - Create OAuth2Client via `createOAuth2Client()` from `@/lib/google/oauth`
   - Set credentials with decrypted tokens: `decrypt(coach.googleAccessToken)`, `decrypt(coach.googleRefreshToken)`, `coach.googleTokenExpiresAt?.getTime()` for expiry_date
   - Listen for `tokens` event on the client to auto-persist refreshed tokens to the database (encrypt new access_token, refresh_token if present, update googleTokenExpiresAt). Use `prisma.coach.update({ where: { id: coach.id }, data: {...} })`. Import prisma from `@/lib/prisma`.
   - Return `google.calendar({ version: "v3", auth: oauth2Client })`

4. Refactor `googleCalendar` object -- same export name but new signatures:
   - `createEvent(coach: CoachWithTokens, params: {...}): Promise<string | null>` -- coach is the FIRST argument now
     - Call `getCoachCalendarApi(coach)`. If null, log warning and return null.
     - Use `coach.googleCalendarId || "primary"` as calendarId
     - Same event structure as before (summary, description, start/end with timezone, location, reminders)
     - Attendees: only include the provided attendees array (do NOT add INSTRUCTOR_EMAIL -- see Research Pitfall 5: coach is inherently the calendar owner/organizer)
     - sendUpdates: "all"
   - `deleteEvent(coach: CoachWithTokens, eventId: string): Promise<boolean>` -- coach is first argument
     - Same pattern: get calendar API, if null return false, delete event using `coach.googleCalendarId || "primary"`
   - `updateEvent(coach: CoachWithTokens, params: {...}): Promise<string | null>` -- coach is first argument
     - Same pattern: get calendar API, if null return null, update event using `coach.googleCalendarId || "primary"`

5. Keep existing `formatAttendee` helper (still useful).

6. All try/catch blocks should log `[CALENDAR]` prefixed messages matching existing convention.

Import `decrypt` and `encrypt` from `@/lib/encryption`, `createOAuth2Client` from `@/lib/google/oauth`, `prisma` from `@/lib/prisma`, `google` from `googleapis`.
  </action>
