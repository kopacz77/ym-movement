<objective>
Extend the existing Phase 17 `RentalsTable` admin surface with: (a) a 3rd top-level tab "Outstanding Payouts" that lists ONLY consigned rentals whose payouts have not been sent, and (b) a per-row "Mark Payout Sent" button (cyan-600) that fires the new `markConsignmentPaidOut` mutation behind a `showConfirmationToast` confirmation. Both changes are additive — the existing Active and Late Fee tabs and their actions (Mark Returned, Release Deposit, Flag Late Fee) stay untouched.

Purpose: This is the admin side of CONSIGN-10 / RENTAL-08. The Outstanding Payouts tab gives admins a single-pane view of "who do I still owe money?" without scrolling through Active and Late Fee tabs. The per-row Mark Payout Sent button works from ANY of the three tabs (wherever the row appears and is eligible), so an admin who's already on the Active tab and notices a rental ready for payout doesn't have to switch tabs to act.

Output: One modified file (zero new files, zero new pages, zero new sidebar entries). After this plan ships, `/admin/wardrobe/rentals` has a 3rd tab and every eligible row has a Mark Payout Sent button.

    ## Edit 3 — Add markPayoutSent mutation hook + handleMarkPayoutSent

    Near the existing mutation hooks (`markReturned`, `releaseDeposit`, `flagLateFee`), add:

    ```tsx
    import { showConfirmationToast } from "@/lib/toast-confirmations";
    import { formatCurrencyFromCents } from "@/lib/utils";
    // (Add these imports only if they aren't already present in the file.)

    const utils = api.useUtils(); // EXISTING — confirm present, don't duplicate

    const markPayoutSent = api.admin.wardrobeRentals.markConsignmentPaidOut.useMutation({
      onSuccess: () => {
        toast.success("Payout marked as sent", {
          description: "The consigner will see the updated status on their earnings tab.",
        });
        utils.admin.wardrobeRentals.listRentals.invalidate();
      },
      onError: (e) => {
        toast.error("Failed to mark payout sent", { description: e.message });
      },
    });

    const handleMarkPayoutSent = (r: RentalRow) => {
      // Guard at the call site too — the button is also conditionally rendered, but
      // belt-and-suspenders for keyboard activation race conditions.
      if (r.consignmentPayoutAmount == null || r.consignmentPaidOut) return;

      showConfirmationToast({
        title: "Mark payout sent",
        description: `Confirm that you have sent ${formatCurrencyFromCents(r.consignmentPayoutAmount)} payout to ${r.Dress.Owner.name ?? "the consigner"} for "${r.Dress.title}"? This action cannot be undone.`,
        confirmLabel: "Mark Sent",
        onConfirm: () => markPayoutSent.mutate({ rentalId: r.id }),
      });
    };
    ```

    If `showConfirmationToast` is invoked differently in the existing file (e.g. positional args vs object form), match the existing style. Per 19-RESEARCH §UI Surfaces, the object form is the precedent in the broader codebase — use it unless the file already has a different convention.
