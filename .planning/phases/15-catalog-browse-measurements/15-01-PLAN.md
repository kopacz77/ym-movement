   Add a `// CAT-08: internalNotes NEVER. consignmentCommissionPct NEVER.` comment directly above the closing brace.

2. `catalogRouter = createTRPCRouter({ list, byId, facets })`:

   **list** — `protectedProcedure.input(catalogFilterSchema).query(...)`:
   - Build `where: Prisma.DressWhereInput = { status: { in: ["AVAILABLE", "PENDING"] } }` (CAT-01)
   - If `input.categories?.length`, add `category: { in: input.categories }`
   - If `input.colors?.length`, add `color: { in: input.colors }`
   - If `input.sizeLabels?.length`, add `sizeLabel: { in: input.sizeLabels }`
   - If `input.themeQuery` (non-empty after trim), add `themeTags: { hasSome: [input.themeQuery] }` (exact-tag-match MVP per research Pattern 4 + Open Question 2)
   - If `input.lengthCmMin != null`, add `lengthCm: { gte: input.lengthCmMin }`; same for `lte` from lengthCmMax (merge into single object if both)
   - If `input.priceMinCents != null`, add `competitionPrice: { gte: input.priceMinCents }`; same for lte (merge)
   - If `input.availableFrom && input.availableTo`, add the two `NOT EXISTS` clauses via `AND: [{ Rentals: { none: {...} } }, { Requests: { none: {...} } }]` per research Pattern 5:
     - Rentals: `paymentStatus: { in: ["AWAITING_PAYMENT", "PAID"] }, startDate: { lte: end }, endDate: { gte: start }`
     - Requests: `status: "APPROVED", startDate: { lte: end }, endDate: { gte: start }`
   - Look up caller measurements (`ctx.prisma.student.findUnique({ where: { userId: ctx.session.user.id }, select: { chestCm, waistCm, hipsCm, heightCm, ... } })`) — null if no Student row
   - If `input.sort === "bestFit"`, throw `TRPCError { code: "BAD_REQUEST", message: "Set chest, waist, or hips before sorting by best fit" }` when `callerMeasurements == null` OR all three of chest/waist/hips are null. Defense-in-depth for CAT-05.
   - If `input.fitsMe === true`, throw `TRPCError { code: "BAD_REQUEST" }` if no chest/waist/hips set (same gating rule)
   - Run `ctx.prisma.dress.findMany({ where, select: PUBLIC_DRESS_SELECT, orderBy: { createdAt: "desc" } })` — fetch full filtered set (bounded by AVAILABLE+PENDING; in-memory sort/paginate after)
   - **Do NOT call `scoreDress` here** — Plan 15-02 ships fitScore.ts. Implementation note: leave a `// TODO(15-02): once fitScore.ts exists, sort+filter using scoreDress/passesFitsMeFilter` comment AND make the sort dispatch work for the 3 non-bestFit cases (newest is the orderBy default; priceAsc/Desc sort in-memory by competitionPrice).
   - For Wave 1 ship, if `sort=bestFit` or `fitsMe=true` reaches the data path (not blocked by the BAD_REQUEST gate above because caller HAS measurements), fall back to `sort: "newest"` ordering and add a `// TODO(15-02)` marker. This keeps the procedure deployable before Plan 15-02 lands.
   - Pagination: in-memory `result.slice((page-1) * limit, page * limit)`. Return `{ items, total, page, limit, callerHasMeasurements }`.
   - `callerHasMeasurements` is a boolean derived from `(callerMeasurements?.chestCm ?? callerMeasurements?.waistCm ?? callerMeasurements?.hipsCm) != null`. Client uses this to gate the Best Fit / Fits Me UI.

   **byId** — `protectedProcedure.input(z.object({ id: z.string().cuid() })).query(...)`:
   - `ctx.prisma.dress.findUnique({ where: { id: input.id }, select: PUBLIC_DRESS_SELECT })`
   - Throw `NOT_FOUND` if null OR if `status NOT IN ["AVAILABLE", "PENDING"]` (browse-only; do not leak existence of ARCHIVED/REJECTED dresses)
   - Return the dress

   **facets** — `protectedProcedure.query(...)`:
   - Run TWO Prisma queries against AVAILABLE+PENDING dresses:
     - `colors: await ctx.prisma.dress.findMany({ where: { status: { in: PUBLIC_STATUSES } }, distinct: ["color"], select: { color: true }, orderBy: { color: "asc" } })`
     - `sizeLabels: await ctx.prisma.dress.findMany({ where: { status: { in: PUBLIC_STATUSES } }, distinct: ["sizeLabel"], select: { sizeLabel: true }, orderBy: { sizeLabel: "asc" } })`
   - Return `{ colors: colors.map(c => c.color), sizeLabels: sizeLabels.map(s => s.sizeLabel) }`
