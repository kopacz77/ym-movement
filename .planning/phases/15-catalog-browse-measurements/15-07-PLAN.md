**Header comment:**
"URL-state-driven catalog grid. Mirrors the Plan 14-05 DressInventoryGrid pattern wholesale per research Pattern 7. Owns:
- URL query params for categories, colors, sizeLabels, themeQuery, lengthCmMin/Max, priceMinCents/Max, availableFrom/To, fitsMe, sort, page

Reads via api.wardrobe.list.useQuery(parsedFilters)."

**Parsing helpers (defined inside the file, mirror 14-05):**
- `parseEnumArrayParam<T>(raw, allowed): T[]` — split on comma, trim, filter to allowed enum values, return []
- `parseStringArrayParam(raw): string[]` — split on comma, trim, drop empty entries
- `parseIntParam(raw): number | undefined` — `Number.parseInt(raw, 10)`, NaN/negative → undefined
- `parseDateParam(raw): Date | undefined` — `new Date(raw)`, invalid → undefined
- `parseBoolParam(raw): boolean` — `raw === "1"`
- `parseSortParam(raw): SortOption` — `sortOptionSchema.catch("newest").parse(raw)`

**Inside CatalogGrid component:**

1. `const router = useRouter()` / `const searchParams = useSearchParams()`

2. Parse all params via `useMemo([searchParams])`:
```ts
const filters = useMemo(() => ({
  categories: parseEnumArrayParam<DressCategory>(searchParams.get("categories"), Object.values(DressCategory)),
  colors: parseStringArrayParam(searchParams.get("colors")),
  sizeLabels: parseStringArrayParam(searchParams.get("sizes")),
  themeQuery: searchParams.get("q") ?? "",
  lengthCmMin: parseIntParam(searchParams.get("lengthMin")),
  lengthCmMax: parseIntParam(searchParams.get("lengthMax")),
  priceMinCents: parseIntParam(searchParams.get("priceMin")),
  priceMaxCents: parseIntParam(searchParams.get("priceMax")),
  availableFrom: parseDateParam(searchParams.get("from")),
  availableTo: parseDateParam(searchParams.get("to")),
  fitsMe: parseBoolParam(searchParams.get("fitsMe")),
  sort: parseSortParam(searchParams.get("sort")),
  page: parseIntParam(searchParams.get("page")) ?? 1,
}), [searchParams]);
```

3. `const { data, isLoading, error } = api.wardrobe.list.useQuery(filters)` — TRPC infers input shape from catalogFilterSchema.

4. `updateParams(patch: Partial<Record<string, string | null>>, resetPage = true)` helper:
```ts
const updateParams = (patch: Record<string, string | null>, resetPage = true) => {
  const next = new URLSearchParams(searchParams.toString());
  for (const [k, v] of Object.entries(patch)) {
    if (v == null || v === "") next.delete(k);
    else next.set(k, v);
  }
  if (resetPage) next.delete("page");
  const qs = next.toString();
  router.replace(qs ? `?${qs}` : "?", { scroll: false });
};
```

5. Wire all 11 change callbacks (filter values + clearAll + page) to `updateParams` calls. Examples:
   - `onCategoriesChange = (next) => updateParams({ categories: next.length ? next.join(",") : null })`
   - `onColorsChange = (next) => updateParams({ colors: next.length ? next.join(",") : null })`
   - `onSizeLabelsChange = (next) => updateParams({ sizes: next.length ? next.join(",") : null })`
   - `onThemeQueryChange = (next) => updateParams({ q: next.trim() ? next.trim() : null })`
   - `onLengthCmChange = (min, max) => updateParams({ lengthMin: min?.toString() ?? null, lengthMax: max?.toString() ?? null })`
   - `onPriceCentsChange = (min, max) => updateParams({ priceMin: min?.toString() ?? null, priceMax: max?.toString() ?? null })`
   - `onAvailabilityChange = (from, to) => updateParams({ from: from?.toISOString().slice(0,10) ?? null, to: to?.toISOString().slice(0,10) ?? null })` (YYYY-MM-DD for clean URLs)
   - `onFitsMeChange = (next) => updateParams({ fitsMe: next ? "1" : null })`
   - `onSortChange = (next) => updateParams({ sort: next === "newest" ? null : next }, /*resetPage*/ false)` — sort changes DO NOT reset page (research Pattern 7 / 14-05 ADR)
   - `onClearAll = () => router.replace("?", { scroll: false })`
   - Pagination: `onPrev` / `onNext` → `updateParams({ page: newPage === 1 ? null : newPage.toString() }, /*resetPage*/ false)` — keep filter context
