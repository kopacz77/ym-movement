# Phase 15 Plan 07: CatalogGrid Summary

**One-liner:** Phase 15 keystone shipped — `/wardrobe` is now a URL-state-driven catalog browse surface composing the WardrobeFilterBar + DressCard grid + paginated footer, replacing the 14-07 Coming Soon stub and unlocking every Phase 15 success criterion.

## What shipped

**`CatalogGrid.tsx`** (382 lines) — URL-state owner that:
- Parses 11 query params (categories, colors, sizes, q, lengthMin/Max, priceMin/Max, from, to, fitsMe, sort, page) on every searchParams change via `useMemo`
- Renders a fully-controlled `<WardrobeFilterBar />` driven by parsed state + change callbacks that route through a centralized `updateParams(patch, resetPage?)` helper
- Renders `<DressCard />` per result item with `fitScorePercent` flowing straight from `wardrobe.list`
- Includes brand-consistent loading skeleton (DressCard-shape placeholders), empty state ("No dresses match your filters"), and a BAD_REQUEST CTA pointing to `/wardrobe/measurements` for the stale-URL Fits Me / Best Fit case
- Pagination footer that preserves filter context (page param is the only one mutated)

**`page.tsx`** — thin 3-line client shell rendering `<CatalogGrid />` (matches 14-06 ADR). Coming Soon stub copy fully gone.

## URL params + elision rules (single-table reference)

| Param | Default | Elision rule |
|---|---|---|
| `categories` | `[]` | empty → delete |
| `colors` | `[]` | empty → delete |
| `sizes` | `[]` | empty → delete |
| `q` | `""` | empty/whitespace → delete |
| `lengthMin` / `lengthMax` | `null` | `null` → delete (edge-of-range thumbs commit as null per 15-06) |
| `priceMin` / `priceMax` | `null` | `null` → delete (edge-of-range thumbs commit as null per 15-06) |
| `from` / `to` | `null` | YYYY-MM-DD serialization; `null` → delete |
| `fitsMe` | `false` | `false` → delete; `true` → `"1"` |
| `sort` | `"newest"` | `"newest"` → delete |
| `page` | `1` | `<=1` → delete |

## Page-reset policy

- **Filter changes** (categories/colors/sizes/q/length/price/dates/fitsMe) → reset page to 1
- **Sort changes** → preserve current page (matches 14-05 Pattern 7 ADR: sort is a reordering of the same result set)
- **Pagination Prev/Next** → preserve all other filters; only page mutates
