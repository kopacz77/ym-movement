  <name>Task 1: Create coach signup API endpoint</name>
  <files>src/app/api/auth/coach-signup/route.ts</files>
  <action>
    Create `src/app/api/auth/coach-signup/route.ts` by adapting the student signup at `src/app/api/auth/signup/route.ts`.

    **Zod schema for coach signup:**
    ```typescript
    const coachSignupSchema = z.object({
      name: z.string().min(1, "Name is required").max(100, "Name too long"),
      email: z.string().email("Invalid email format").max(255, "Email too long"),
      phone: z.string().optional(),
      bio: z.string().max(500, "Bio too long").optional(),
      skills: z.array(z.string()).optional().default([]),
      certifications: z.string().max(1000, "Certifications too long").optional(),
      yearsExperience: z.number().int().min(0).max(99).optional(),
      honeypot: z.string().optional(),
      turnstileToken: z.string().optional(),
    });
    ```

    **POST handler pattern (copy from student signup route):**
    1. Rate limiting using `authRateLimiter` (same as student -- 5 per IP per hour)
    2. Parse and validate body with `coachSignupSchema`
    3. Honeypot check (return generic error if filled)
    4. Turnstile token verification (same `verifyTurnstileToken` function -- copy it or extract to shared utility)
    5. Format inputs: `toProperCase(name)`, `formatEmail(email)`, `formatPhoneNumber(phone)`
    6. Check for duplicate email: `prisma.user.findUnique({ where: { email } })`
    7. Transaction to create User + Coach:
       ```typescript
       const user = await prisma.$transaction(async (tx) => {
         const newUser = await tx.user.create({
           data: {
             name,
             email,
             password: null,  // Password set during registration completion after approval
             role: "COACH",
           },
         });

         const coach = await tx.coach.create({
           data: {
             userId: newUser.id,
             bio: bio || null,
             skills: skills || [],
             certifications: certifications || null,
             yearsExperience: yearsExperience ?? null,
             isApproved: false,
             isActive: false,  // Not active until approved
           },
         });

         return { ...newUser, Coach: coach };
       });
       ```
    8. Send welcome email (non-blocking, don't fail if email fails). Use existing `sendWelcomeEmail` or create a `sendCoachApplicationEmail` if desired. For now, reuse `sendWelcomeEmail` with the coach's email and name.
    9. Return 201 with success message (exclude password from response)

    **Imports needed:**
    - `z` from "zod"
    - `prisma` from "@/lib/prisma"
    - `authRateLimiter`, `getClientIP`, `logSecurityEvent`, `turnstileTokenTracker` from "@/lib/security"
    - `formatEmail`, `formatPhoneNumber`, `toProperCase` from "@/lib/utils"
    - `sendWelcomeEmail` from "@/lib/email"

    Copy the `verifyTurnstileToken` function from the student signup route directly into this file (avoid modifying the shared student signup file).

    IMPORTANT: The Coach record starts with `isApproved: false` AND `isActive: false`. It becomes active only after admin approval.
