  <action>
    Create the directory `src/features/coach/api/queries/`.

    **1. dashboardQueries.ts** (CDSH-01):
    Create a router with `coachProcedure` from `@/lib/trpc`:

    - `getUpcomingLessons`: query with optional `limit` input (default 10, max 50). Fetch lessons where `coachId: ctx.coach.id`, `startTime >= now()`, `status: "SCHEDULED"`. Include `Student.User` (name, email), `Rink` (name, timezone). Order by startTime asc. Also include `type` and `price` fields from Lesson.

    - `getPastLessons`: query with optional `limit` input (default 10, max 50). Fetch lessons where `coachId: ctx.coach.id`, `startTime < now()`. Include `Student.User` (name, email), `Rink` (name). Order by startTime desc.

    - `getDashboardStats`: query returning counts:
      - Total students (count distinct studentId from Lesson where coachId)
      - Upcoming lessons count
      - This month's completed lessons count
      - Total earnings this month (sum of payments where Lesson.coachId and status COMPLETED and lesson_date >= startOfMonth)

    **2. profileQueries.ts** (CMGT-02):
    - `getProfile`: coachProcedure query. Fetch coach by `ctx.coach.id` including `User` (name, email, createdAt). Return full coach profile.

    - `updateProfile`: coachProcedure mutation with Zod input:
      - bio: z.string().max(500).optional()
      - photoUrl: z.string().url().optional().nullable()
      - skills: z.array(z.string()).optional()
      - certifications: z.string().max(1000).optional().nullable()
      - yearsExperience: z.number().int().min(0).max(99).optional().nullable()
      Update coach record. Do NOT allow editing pricing fields (coach can only view rates, per REQUIREMENTS.md "Coach-set pricing" is out of scope).

    **3. earningsQueries.ts** (CDSH-03):
    - `getEarningsSummary`: coachProcedure query. Use Promise.all for parallel aggregation:
      - Total all-time earnings: sum payments where Lesson.coachId and status COMPLETED
      - This month earnings: same but lesson_date >= startOfMonth
      - Pending payments: sum + count where status PENDING
      - Apply revenueSplitPercent from ctx.coach (divide by 100)
      - Return totalEarnings, monthEarnings, pendingAmount, pendingCount, revenueSplitPercent

    - `getPaymentHistory`: coachProcedure query with optional limit (default 20). Fetch payments where Lesson.coachId, include Student.User (name), lesson date, amount, status, paymentMethod. Order by createdAt desc.

    **4. studentQueries.ts** (CDSH-04):
    Export as `coachStudentsRouter` (not `studentRouter` to avoid name collision with existing student router).

    - `getMyStudents`: coachProcedure query. Fetch CoachStudent records where coachId: ctx.coach.id, include Student.User (name, email), Student (level, isActive, isApproved). Also count each student's total lessons with this coach. Order by createdAt desc.

    **5. index.ts**:
    ```typescript
    import { createTRPCRouter } from "@/lib/trpc";
    import { dashboardRouter } from "./dashboardQueries";
    import { profileRouter } from "./profileQueries";
    import { earningsRouter } from "./earningsQueries";
    import { coachStudentsRouter } from "./studentQueries";

    export const coachRouter = createTRPCRouter({
      dashboard: dashboardRouter,
      profile: profileRouter,
      earnings: earningsRouter,
      students: coachStudentsRouter,
    });
    ```

    Note: proposalRouter will be added in Plan 06.

    **6. root.ts**:
    Add import: `import { coachRouter } from "@/features/coach/api/queries/index";`
    Add to appRouter: `coach: coachRouter,`

    IMPORTANT:
    - Always use PascalCase for Prisma relations: Student.User, Lesson.Student, Lesson.Rink
    - Always scope by ctx.coach.id -- never return unscoped data
    - Import coachProcedure from @/lib/trpc (not protectedProcedure)
    - Use `lesson_date` (snake_case) for the payment date field -- check actual Prisma schema field name
