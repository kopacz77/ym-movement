    **1. coachApprovalQueries.ts** (CMGT-01):
    Mirror the pattern from `src/features/admin/api/queries/student/approvalQueries.ts` exactly.

    Use `superAdminProcedure` (not protectedProcedure) for all admin coach management queries.

    - `getPendingCoachApprovals`: query. Fetch coaches where `isApproved: false`, include User (name, email, createdAt). Return formatted array with id, user info, status, createdAt. Also include bio, skills, certifications, yearsExperience for the admin to review the application.

    - `approveCoach`: mutation, input `{ coachId: z.string() }`.
      1. Find coach by id with User include
      2. Update coach: `isApproved: true, isActive: true, approvedAt: new Date(), approvedById: ctx.session.user.id`
      3. Create password reset token using `createPasswordResetToken` (import from `@/lib/auth-tokens`)
      4. Send approval email using `sendApprovalEmail` (import from `@/lib/email`). If no specific coach approval email exists, reuse `sendApprovalEmail` which generates a registration completion link.
      5. Return updated coach

    - `denyCoach`: mutation, input `{ coachId: z.string() }`.
      1. Find coach by id with User include
      2. Delete the Coach record
      3. Delete the User record (or just the Coach if you want to keep the user)
      4. Return success message
      Note: Follow the same pattern as student `rejectStudent` -- delete the Coach record. The User can be kept or deleted depending on the cascade behavior in the schema. Check if Coach has `onDelete: Cascade` on the User relation. The Coach model has `User @relation(fields: [userId], references: [id], onDelete: Cascade)` meaning deleting the User cascades to Coach. But we want to delete both -- so delete the User, which cascades to Coach.

    **2. coachManagementQueries.ts** (CMGT-03, CMGT-04):

    - `getAllCoaches`: query. Fetch all coaches including User (name, email), ordered by createdAt desc. Return with status info (isActive, isApproved, suspendedAt).

    - `getCoachById`: query, input `{ coachId: z.string() }`. Fetch single coach with all details + User + lesson count + student count.

    - `createCoach`: mutation (CMGT-03). Input:
      ```typescript
      z.object({
        name: z.string().min(1).max(100),
        email: z.string().email(),
        bio: z.string().max(500).optional(),
        skills: z.array(z.string()).optional().default([]),
        certifications: z.string().max(1000).optional(),
        yearsExperience: z.number().int().min(0).max(99).optional(),
        privateLessonPrice: z.number().min(0).optional(),
        groupLessonPrice: z.number().min(0).optional(),
        choreographyPrice: z.number().min(0).optional(),
        competitionPrepPrice: z.number().min(0).optional(),
        revenueSplitPercent: z.number().min(0).max(100).optional().default(70),
      })
      ```
      Transaction:
      1. Check for duplicate email
      2. Create User with role: "COACH", password: null
      3. Create Coach with isApproved: true, isActive: true, approvedAt: now, approvedById: ctx.session.user.id, plus all provided fields
      4. Create password reset token and send email for registration completion
      5. Return created coach

    - `updateCoachPricing`: mutation. Input: `{ coachId, privateLessonPrice?, groupLessonPrice?, choreographyPrice?, competitionPrepPrice?, revenueSplitPercent? }`. Update coach pricing fields. This is how the admin sets/changes coach rates.

    - `toggleCoachStatus`: mutation (CMGT-04). Input:
      ```typescript
      z.object({
        coachId: z.string(),
        action: z.enum(["activate", "deactivate", "suspend"]),
        reason: z.string().optional(),
      })
      ```
      Switch on action:
      - activate: `isActive: true, suspendedAt: null, suspendedById: null, suspendedReason: null`
      - deactivate: `isActive: false`
      - suspend: `isActive: false, suspendedAt: new Date(), suspendedById: ctx.session.user.id, suspendedReason: input.reason ?? null`
