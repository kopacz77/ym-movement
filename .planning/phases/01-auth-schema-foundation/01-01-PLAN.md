  <action>
Update the JWT callback in `src/lib/auth.ts` (lines 122-137) to handle the `trigger === "update"` case. This allows the frontend to call `update()` on the session to force a role refresh from the database, so Yura doesn't need to log out and back in when her role changes from ADMIN to SUPER_ADMIN.

Current callback (line 123):
```typescript
async jwt({ token, user }) {
  if (user) {
    token.id = user.id;
    token.role = user.role;
  }
  return token;
},
```

Updated callback:
```typescript
async jwt({ token, user, trigger }) {
  if (user) {
    token.id = user.id;
    token.role = user.role;
  }
  // On session update trigger, refresh role from database
  // This allows role changes (e.g., ADMIN -> SUPER_ADMIN) to propagate without re-login
  if (trigger === "update") {
    const dbUser = await prisma.user.findUnique({
      where: { id: token.id as string },
      select: { role: true },
    });
    if (dbUser) {
      token.role = dbUser.role;
    }
  }
  return token;
},
```

The `prisma` import is already available in this file (imported on line 6).

Do NOT modify any other part of auth.ts (the authorize function, session callback, providers, or options).
