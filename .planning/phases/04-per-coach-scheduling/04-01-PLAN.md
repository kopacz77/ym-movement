  <action>
    Create `src/features/coach/api/queries/scheduleQueries.ts` with a `scheduleRouter` containing 6 endpoints:

    1. **getMyTimeSlots** (coachProcedure, query): Fetches time slots for `ctx.coach.id`.
       - Input: `{ startDate?: Date, endDate?: Date, rinkId?: string }`
       - Where: `coachId: ctx.coach.id`, `isActive: true`, optional rinkId and date range filters
       - Select: Mirror the admin `getTimeSlots` select shape EXACTLY (id, rinkId, startTime, endTime, maxStudents, isActive, createdAt, Rink with id/name/timezone/address, Lesson with id/type/price/status/notes/Student with id/notes/User with id/name/email, StudentNote with id/content/createdAt/type/User.name)
       - OrderBy: `{ startTime: "asc" }`

    2. **getMyBlockedDates** (coachProcedure, query): Fetches blocked dates for `ctx.coach.id`.
       - Input: `{ startDate?: Date, endDate?: Date }`
       - Where: `coachId: ctx.coach.id`, with date range overlap logic (same as admin blockedDateQueries.ts getBlockedDates)
       - Include: `User: { select: { id: true, name: true, email: true } }`
       - OrderBy: `[{ startDate: "asc" }]`

    3. **getRinks** (coachProcedure, query): Fetches all rinks (coaches need rink data for schedule display).
       - No input required.
       - Select: `{ id: true, name: true, timezone: true, address: true }`
       - This mirrors the pattern from proposalQueries.ts which already has getRinks for coaches.

    4. **createBlockedDate** (coachProcedure, mutation): Creates a blocked date for the coach.
       - Input: `{ title: string (min 1, max 100), description?: string, startDate: Date, endDate: Date, type: enum TRAVEL/COMPETITION/OTHER default TRAVEL }` with refine ensuring endDate >= startDate
       - Auto-assign `coachId: ctx.coach.id` and `createdById: ctx.session.user.id`
       - Return success message and created record

    5. **updateBlockedDate** (coachProcedure, mutation): Updates an existing blocked date owned by the coach.
       - Input: `{ id: string, title?: string (min 1, max 100), description?: string, startDate?: Date, endDate?: Date, type?: enum TRAVEL/COMPETITION/OTHER }` with refine ensuring endDate >= startDate when both provided
       - First verify the blocked date exists AND belongs to `ctx.coach.id` (throw FORBIDDEN if not)
       - Check for overlapping ranges (excluding current record, scoped to `ctx.coach.id`) when dates change -- same overlap logic as admin `updateBlockedDate` in `blockedDateQueries.ts`
       - Update and return success message + updated record
       - Model this endpoint closely on the admin `updateBlockedDate` mutation in `blockedDateQueries.ts`, but replace `adminProcedure` with `coachProcedure` and enforce ownership via `ctx.coach.id`

    6. **deleteBlockedDate** (coachProcedure, mutation): Deletes a blocked date owned by the coach.
       - Input: `{ id: string }`
       - First verify the blocked date exists AND belongs to `ctx.coach.id` (throw FORBIDDEN if not)
       - Delete and return success message

    Use the same code patterns as `proposalQueries.ts` and `earningsQueries.ts` for error handling (try/catch with TRPCError). Import `coachProcedure` and `createTRPCRouter` from `@/lib/trpc`, `z` from `zod`, `TRPCError` from `@trpc/server`.

    Then update `src/features/coach/api/queries/index.ts` to import `scheduleRouter` from `./scheduleQueries` and add `schedule: scheduleRouter` to the `coachRouter`.
