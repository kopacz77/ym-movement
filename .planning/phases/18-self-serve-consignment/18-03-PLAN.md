<objective>
Refactor the existing 626-line `DressForm.tsx` into a reusable `DressFormCore` plus two thin wrappers: `DressForm` (admin, all fields visible) and `ConsignerDressForm` (consigner, admin-only fields hidden + pricing/size locked after first approval). DressFormCore is the single source of truth for field rendering, dollar↔cents conversion, and Zod validation.

Purpose: Without this refactor, ConsignerDressForm would be a 600-line fork of DressForm and bug fixes would diverge. Per 18-RESEARCH Critical Findings §5, the recommended approach is Option B (extract core). This plan extracts the core and makes both forms thin wrappers, paying immediate dividends in Plan 18-06 (consigner edit page composition) and in Phase 21 testing (one test surface covers both).

Output: Three component files. `DressForm.tsx` shrinks from ~626 lines to ~30 lines (preserving its public prop shape — no caller change needed in `/admin/wardrobe/new` or `/admin/wardrobe/[id]/edit`). `DressFormCore.tsx` contains the bulk of the rendering logic (~450 lines extracted near-verbatim from the existing DressForm). `ConsignerDressForm.tsx` is a new ~50-line wrapper that passes `fieldVisibility` hiding admin fields and `fieldLocking` driven by `dress.status`.
</objective>

    Replace the entire contents of `src/features/wardrobe/components/admin/DressForm.tsx` with a ~30-line wrapper:

    ```ts
    // src/features/wardrobe/components/admin/DressForm.tsx
    //
    // Admin DressForm — thin wrapper around DressFormCore.
    //
    // Admins see ALL fields and have ZERO field locking. The actual rendering logic
    // lives in DressFormCore so the ConsignerDressForm (Plan 18-03 sibling) can
    // share the same field plumbing with different visibility/locking.

    "use client";

    import {
      DressFormCore,
      type FieldLocking,
      type FieldVisibility,
      type DressFormValues, // re-exported below
    } from "@/features/wardrobe/components/DressFormCore";
    import type { DressInput } from "@/features/admin/api/queries/wardrobeDressQueries";

    const ADMIN_VISIBILITY: FieldVisibility = {
      showInternalNotes: true,
      showCommissionPct: true,
      showSecurityDepositAndCleaning: true,
      showStatusSelect: true,
    };

    const ADMIN_LOCKING: FieldLocking = {
      lockPricingAndSize: false,
    };

    interface DressFormProps {
      mode: "create" | "edit";
      defaultValues?: Partial<DressFormValues>;
      onSubmit: (input: DressInput & { status?: import("@prisma/client").DressStatus }) => void;
      isSubmitting: boolean;
      submitLabel?: string;
    }

    export function DressForm(props: DressFormProps) {
      return (
        <DressFormCore
          {...props}
          fieldVisibility={ADMIN_VISIBILITY}
          fieldLocking={ADMIN_LOCKING}
        />
      );
    }

    // Re-export DressFormValues so existing callers (e.g. /admin/wardrobe/[id]/edit
    // mapping the dress to defaultValues) keep working without import path churn.
    export type { DressFormValues };
    ```

    ```ts
    // src/features/wardrobe/components/consigner/ConsignerDressForm.tsx
    //
    // Consigner-facing dress form — thin wrapper around DressFormCore.
    //
    // Per CONSIGN-02:
    //   - internalNotes hidden (admin-only)
    //   - consignmentCommissionPct hidden (server pulls from Settings)
    //   - securityDeposit + cleaningFee hidden (server uses schema defaults)
    //   - status select hidden (server forces PENDING_APPROVAL on create; never
    //     changed by consigner update — only by admin or by consigner.resubmit)
    //
    // Per CONSIGN-04:
    //   - Pricing + size editable in PENDING_APPROVAL and REJECTED states
    //   - Pricing + size LOCKED in all other states (parent computes
    //     lockPricingAndSize from dress.status and passes through)

    "use client";

    import type { DressStatus } from "@prisma/client";
    import {
      DressFormCore,
      type FieldLocking,
      type FieldVisibility,
      type DressFormValues,
    } from "@/features/wardrobe/components/DressFormCore";

    const CONSIGNER_VISIBILITY: FieldVisibility = {
      showInternalNotes: false,
      showCommissionPct: false,
      showSecurityDepositAndCleaning: false,
      showStatusSelect: false,
    };

    interface ConsignerDressFormProps {
      mode: "create" | "edit";
      defaultValues?: Partial<DressFormValues>;
      /**
       * Submit handler receives the cents-based DressInput (post-conversion).
       * On consigner side this maps to `wardrobe.consigner.{create,update}`. The
       * parent page is responsible for stripping the locked keys on update if the
       * dress is post-approval — but the server is the authority and will
       * BAD_REQUEST anyway.
       */
      onSubmit: (input: import("@/features/admin/api/queries/wardrobeDressQueries").DressInput) => void;
      isSubmitting: boolean;
      submitLabel?: string;
      /**
       * Computed by the parent page from dress.status:
       *   const lockPricingAndSize = dress.status !== "PENDING_APPROVAL" && dress.status !== "REJECTED";
       * Default false (suitable for create mode, where the dress doesn't exist yet
       * so cannot be post-approval).
       */
      lockPricingAndSize?: boolean;
    }

    export function ConsignerDressForm({
      mode,
      defaultValues,
      onSubmit,
      isSubmitting,
      submitLabel,
      lockPricingAndSize = false,
    }: ConsignerDressFormProps) {
      const fieldLocking: FieldLocking = { lockPricingAndSize };
      return (
        <DressFormCore
          mode={mode}
          defaultValues={defaultValues}
          fieldVisibility={CONSIGNER_VISIBILITY}
          fieldLocking={fieldLocking}
          onSubmit={onSubmit}
          isSubmitting={isSubmitting}
          submitLabel={submitLabel ?? (mode === "create" ? "Save & continue" : "Save changes")}
        />
      );
    }
    ```

    Note on submit handler typing: ConsignerDressForm's `onSubmit` receives the cents-converted DressInput. The parent page (Plan 18-06) maps this to `wardrobe.consigner.create.mutate(input)` or `wardrobe.consigner.update.mutate({ id, ...input })`. The server's Zod `.pick()` (Plan 18-02) silently strips any locked fields that snuck through on a post-approval edit — defense-in-depth.

    Note on default submit label: `mode === "create"` produces "Save & continue" (mirrors Phase 14-06 create-then-redirect copy); `mode === "edit"` produces "Save changes" (mirrors admin DressForm).
