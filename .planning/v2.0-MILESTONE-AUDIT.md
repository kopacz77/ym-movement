## User Flow Walks: 4/4

All 4 critical user flows traced through the codebase with every step backed by a real procedure or page:

**1. Student rental happy path** — `/wardrobe` browse → `/wardrobe/[id]` detail → RequestRentalDialog → PENDING → `/admin/wardrobe/requests` admin approves → `/admin/wardrobe/rentals` mark paid → mark returned → release deposit. Code path verified end-to-end.

**2. Consigner happy path** — `/wardrobe/consigned/new` → PENDING_APPROVAL → admin approves → AVAILABLE → visible in catalog. Code path verified.

**3. Consigner rejection + resubmit** — Admin rejects with reason → consigner sees banner → edits + resubmits → admin approves on second pass. Code path verified.

**4. Permission gates (PERM-01..04)** — All 4 negative paths have code-level enforcement (consigner ownership via `assertOwnsDress`, admin gates via `adminProcedure`, caller-owns guards on rental requests, E2E negative-path tests covering both HTTP 401/403 and TRPC error envelopes).

All 9 lifecycle notifications fire from the correct mutation call site in non-blocking try/catch wrappers so a Resend outage cannot block the parent mutation or suppress the in-app notification.
