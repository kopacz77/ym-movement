## E2E Flow Verification

### Flow 1: New Coach Onboarding
**Status:** PASSED (after middleware fix)

Signup → Admin approval → Calendar connect → Propose slots → Admin approves → Students can book

- Coach signup page accessible at `/auth/coach-signup` (fixed during audit)
- API route creates User + Coach in $transaction
- Admin pending approvals queue shows new applications
- Approval sends email, activates account
- Coach connects Google Calendar via OAuth
- Coach proposes time slots, admin approves creating RinkTimeSlots

### Flow 2: Student Booking
**Status:** PASSED

Browse coaches → Select coach → View slots → Book → Coach notified → Calendar event → Payment tracked

- CoachBrowse grid shows approved/active coaches
- Two-step flow: coach selection then calendar with coachId filter
- bookLesson inherits coachId from timeSlot, creates lesson + payment
- Per-coach Google Calendar event created (or graceful skip)
- Coach receives SUCCESS notification
- CoachStudent junction record upserted

### Flow 3: Admin Revenue Management
**Status:** PASSED

Set split % → View payouts → Export CSV → Revenue breakdown correct

- Inline RevenueSplitCell editor in CoachList
- updateCoachPricing mutation notifies coach of change
- PayoutReport shows per-coach breakdown with correct calculations
- CSV export includes headers, per-coach rows, TOTAL row
- Dashboard RevenueBreakdownChart shows platform-wide and per-coach data

### Flow 4: Dual-Role Owner (Yura)
**Status:** PASSED

Admin view → "Coach View" link → Coach dashboard → "Admin View" link → Back to admin

- useCurrentUser fetches Coach record for SUPER_ADMIN/ADMIN
- AppSidebar shows "Coach View" when coachId present
- Middleware allows SUPER_ADMIN on /coach/* routes
- Coach queries scope by ctx.coach.id (sees own data only)
- AppSidebar shows "Admin View" when isAdmin true
